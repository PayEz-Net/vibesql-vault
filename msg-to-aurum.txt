## What vsql-vault IS

A governed opaque storage vault for pre-encrypted data. It does NOT encrypt or decrypt — it receives already-encrypted blobs and wraps them with:
- Access policies (who can store/retrieve/purge)
- Retention policies (max TTL, default TTL, auto-expiry)
- Audit logging (every operation logged)
- Purge proof (SHA-256 evidence of deletion for compliance)

Think of it as a compliance wrapper around encrypted data, targeting PCI DSS v4.0 Requirement 3.

## Architecture at a Glance

- Rust workspace: vsql-vault-core (types, traits, memory backend) + vsql-vault-server (axum HTTP API, PostgreSQL backend)
- axum 0.8 HTTP framework, sqlx 0.8 async Postgres
- FROM scratch Docker image (~10MB binary)
- API key auth with constant_time_eq
- 5 storage tables: vault_entries, access_log, access_policies, retention_policies, purge_log
